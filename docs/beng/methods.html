 Methods and Process

3.1 Section Introduction

This section considers the processes required for the prototype system to deliver the required results. This involves both a schematic description- involving inputs, processes and outputs- and a more detailed description of each of these processes.

Overall Process

3.2.1 Introduction

The prototype must allow the user to identify what the object displacement is from its current location, and that on a stored (archived) image. For this, it needs to know the displacement of the object, in the plane, and in a real-life quantity such as millimetres.

As stated previously (Section 1.3.1), the problem splits broadly into three tasks:

Locating the plane: Positioning the prototype device correctly relative to the object plane.

Locating the object: Finding the position of the object within the fixed boundaries of the area on the plane.

Finding object displacement: Finding how far the object has moved in the plane from its original position.

Tasks 2 and 3 can be handled at the same time: One first has to locate the object in order to determine its disparity, and thus any object image created in task 2 can subsequently be used to determine object displacement. Thus tasks 2 and 3 are combined together, in the code, into what is called Stage 2.

Whereas tasks 2 and 3 are best combined, the task of positioning the prototype device correctly relative to the plane  (Task 1) is best handled if broken down into a number of sub-stages.

Stage 1: Prototype Positioning

Ideally, the camera should be positioned such that it faces the bounded area of the plane directly; and at a distance from the plane far enough that all of the marker features are in view, but near enough that the pixel resolution of the camera device does not greatly impair the ability for the device to determine small degrees of object disparity (as the distance increases, so does the area of real space represented by a single pixel).

When dealing with a simple rectangular pattern of features, as in the test case, the camera should ideally be positioned such that the features are as follows: [Figure — original CorelDRAW graphic not available in web version]

One could perform a very basic position detection system by attempting to determine that the centre of the features is at the centre of the image, and that the position of the features in the image are equidistant from this central point; but this would only tell the user whether the camera is positioned correctly, or not correctly. To aid the user in positioning the device, the system needs to be able to tell the user both whether the camera is positioned correctly or not, and – if not correctly aligned – how the image is mis-positioned, and what action the user should take to rectify this.

There are a number of specific ways that the camera could be mis-positioned. It could be: parallel to the plane but rotated slightly in this plane (1); rotated slightly in the ‘vertical’ plane perpendicular to that of the object plane (2); or rotated slightly in the ‘horizontal’ plane perpendicular to that of the object plane (3): [Figure — original CorelDRAW graphic not available in web version]

Aligning the camera can be handed as a three-stage process. For demonstration purposes, the prototype only works to determine type (1) rotation, and then type (3) rotation. Type (2) rotation is analogous to type (3) rotation, and so showing that the method works for type (3) rotation means that type (2) problems can also be solved.

Thus, the process used to determine whether the camera is correctly positioned assumes the camera is first positioned correctly vertically: that it is at the correct height relative to the plane, and not at an angle. 

Assuming this, the code first corrects type (1) rotation (the Main 1a function), then type (3) rotation (The Main 1b function). A full system would include a third function to determine type (2) rotation, but the process for this is analogous to correcting type (3) rotation, and so the system does not have to demonstrate this in order to demonstrate its effectiveness.

Stage 2: Determining Object Displacement

Once correctly positioned, the next stage is to use digital image correlation (DIC) methods to determine the displacement between the current object position, and that in an archived image. 

Performing a straight DIC comparison of the two images (the current image, and the archive one where the object is correctly positioned) is not an effective way of determining object displacement. A DIC output of a full image may suggest a ‘match’ at a number of positions, for a number of reasons. If there is a repeated pattern on the image plane then the DIC method will match each repeated feature in the pattern in one image, to both the equivalent and the adjacent repeated features in the other image. 

Although Stage 1 of the process ensures the camera is at the right angle relative to the object area, it does not ensure the camera is in a completely central position regarding shifts in the direction of the plane. The camera position of the current image may be displaced horizontally and vertically. To ensure that this form of displacement, known as rigid body displacement, does not get confused by the computer with object displacement, DIC is performed on the marker masks of both the archive and the current images. This produces an unambiguous figure for this rigid body displacement. Using this, versions of the two images – current and archive – are created that compare only the region of the images that are common to both, and in the object intensity plane.

A DIC is then performed of these two adjusted object images, and the displacement figure given by finding the position of the greatest intensity in the DIC output used as the object displacement in pixels.

To convert the displacement in pixels to a displacement in millimetres (or another real unit), a conversion scale is created. This involves finding the distance in pixels between markers, and comparing them with the values given for distance between the same markers as part of the image geometry information given at the start. In the test example- four markers marking out the corners of a rectangle- the distances between the centres of the two ‘rows’ and ‘columns’ of markers are compared with the height and width of the rectangle given in the marker geometry information.
